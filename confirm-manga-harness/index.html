2. claude / codex 同等性はどう担保するか

  • 創作テキスト以外は全部CLI：モデルの仕事は brief / pages.json / visual_prompts.tsv だけ。画像生成・レンダ・QA・サイトは node scripts/manga-*.mjs ＝誰が回しても決定論で同じ結果。
  • 品質はゲートで守る：機械QA → fork独立採点（≧90）→ 人間ゲート の3段。モデルの善意・自己採点に依存しない。
  • 採点も両モデル対応：同じ採点プロンプトを、claudeはまっさらなfork subagentへ、codexは codex exec へ（codex-cli 0.142.5 導入確認済み）。契約違反の採点JSONは自動0点化。

3. 自己改善ループ（2階建て）

  • 内側ループ：1作品を「作る → fork独立採点 → feedbackの箇所だけ直す」で90点まで詰める（creative-loop・threshold 90 / max 3 / 最良版採用）。
  • 外側ループ：完了ごとに教訓を LESSONS.md へ → 機械ゲート・テンプレ・採点表に実装して初めて applied（台帳に書くだけでは closed にならない）。今日だけで L-08（人物ドリフト→機械検査化）と L-10（ループ機構自体のバグ修正）がこの経路で還流済み。
