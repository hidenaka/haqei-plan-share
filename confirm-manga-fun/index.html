HUMAN GATE — 「面白い」を測るハーネスへ昇華

「読めるが面白くない」の正体は、採点表に“面白さ”軸が無かったこと。一次情報で作り直しました

面白い漫画の作り方を一次情報（漫画編集・原作術）で調べ、6つの採点軸＋企画チェックリスト＋「画像を作る前の面白さゲート」に落とし、ループに組み込みました。同じ001でその効きを実証しています。

TL;DR
① 根本原因＝旧ハーネスは「面白い」を一度も測っていなかった（読める・安全・整うだけ）。測らない軸は良くならない。
② 一次情報20件（各3-0検証）から面白さ6軸を設計＝manga-story採点表＋story-craft.md。
③ 実証：同じ001が仕上げ採点89点 / 面白さ採点49点。ギャップが「面白くない」の正体。
④ ループで面白さを49→61→69→75まで機械的に上昇（画像枠ゼロのテキスト改稿）。90は創作方向の判断が要る＝あなたの領分。

1. 一次情報で抽出した「面白さ6軸」

出典（各主張3-0で検証採択）: 少年ジャンプ漫画学校 / Save the Cat / Kishōtenketsu(Wikipedia) / Reedsy / McCloud『Understanding Comics』 / シナリオ・センター

軸一次情報の原則制作チェック
変化 18ジャンプ第一原則「最初と最後で何かが変わる」。変化が小さい＝地味1p目と最終pの差を1文で言えるか
キャラを好きに 18第二原則「話はキャラを好きにさせる手段」。憧れ性×共通性(欠点)＋利他長所と欠点の両方＋誰かのために動くか
欠落と弧 16Orphan＝欠けて始まる／want→needのすり替え(Story Circle)開幕の欠落と、want→needの気づき
転 16Kishōtenketsu「転」＝3/4で主要素を伏せて出し反転。ビートは1-2文で言えて存在見方を反転させる転を1文で言えるか
引き 16週刊連載の思想＝各単位末に未解決の問い・急転で次を読ませる各ページ末が「次どうなる？」を作るか
葛藤と感情 16三幕＝葛藤で駆動。クライマックスにページを厚く、説明台詞を避ける阻む障害と感情の山、ナレ依存でないか

2. 実証：同じ001、二つの採点

仕上げ採点(manga-render) = 89点（読める・安全・整う）
面白さ採点(manga-story) = 49点（面白い）
このギャップこそ「読めるが面白くない」の数値的な正体。旧ループが89で合格させていたものを、新ゲートは49で落とす。面白さは絵でなく企画・ネームのテキストで判定するので、画像を作る前の安い段階で止められる。

3. ループで面白さが機械的に上がることの実証（画像枠ゼロ）

採点者のfeedback（利他が無い・他者不在・転が内省・ページ末が教訓で閉じる…）どおりに脚本だけ直して再採点を反復：

周面白さ点入れた面白さの手
iter149（初版＝教訓型の平板）
iter261→69慕う後輩ハルを登場・利他の瞬間・外的な事件を追加
iter375キャプション全廃→セリフ/行動駆動・転を外的化(トレス束+破り)・無言の崩れページで山

49 → 61 → 69 → 75（+26点）。プラトーでなく一貫上昇＝ループは面白さを上げる装置として機能している。

4. なぜ今 90 でなく、ここで止めて聞くか

残る失点は採点者いわく①転にミスリード→反転が欲しい（トレスで一見“順調”→実は檻だったと裏返す）②ページ末を軟着陸でなく引きで切る③外形変化を絵に出す。これらは物語の創作方向の判断を含み、主人公・後輩という設定ごと owner の好みが要る領域です（企画・台本は人間ゲート）。max3周は登り切る前に切っただけで、あと数周のテキスト改稿（画像枠は不使用）で90に届く見込みです。
