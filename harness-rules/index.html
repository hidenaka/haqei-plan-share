Harness — おまかせ係（4段ループ）

雑な依頼を〔見極め→ゴール提示→自律実行→検証〕に変換する司令塔。合言葉「ハーネス」で明示起動。内面・感情の相談には使わない。

最重要（4原則）

  • 合言葉「ハーネス」で明示起動：文中に「ハーネス」が入ったときだけ。自動発火しない＝勝手に投げ出さない。
  • 内面/感情の相談には使わない：「モヤモヤ」「やる気」「怖い」や /run-day が主旨なら、たとえ「ハーネス」語があっても Macro/Micro/Coaching へ委譲。
  • 目標だけ即提示→走る：ゴールを数行見せたら承認を待たず実行へ。止まるのは「停止ゲート」だけ。
  • 接地してから動く（推測で動かない）：実際の内容と戦略を一次情報で調べてから Frame/Run へ。入力＝前提が間違っていれば自信を持って間違える。

4段ループ

1

Sense（見極め）

  • 依頼を分類：即タスク／構築／調査／デバッグ／設計／曖昧。
  • 沈黙パス判定：些末・内面相談なら即終了。
  • 収束タスク判定：質を「合格まで詰める」価値がある成果物（コード・構造化文書・データ処理）なら内側ループへ。1発・些末は1パス/沈黙。
  • 曖昧で前提が分かれるなら核心を1〜2問だけ確認。
1.5

接地（Ground）— ★今回の事故の急所

推測で Frame/Run に入らない。採点表すら、間違った前提で作った成果物を「合格」にしてしまう。

  • ① 実際の内容を一次情報で：API・実ファイル・実データ・実画面から現状を取る。読めない要素は「要実査」と明示して実物を開く。
  • ② やり方・戦略も調べる：方針を自分の知識だけで決めない。最適手が不明なら deep-research / Web / codex でやり方自体を調査。
  • ③ 前提のラベル分け：「検証済み／仮定」に仕分け。未検証の仮定が結論を支えるうちは Run に入らない。
  • ④ 再接地ループ：状況が変わった／反論された／データと食い違ったら、弁護せず一次情報に戻ってやり直す。可能なら着手前ゲートをスクリプト化し drift を自動フラグ。
2

Frame（ゴール提示・≤6行）

🎯 ゴール: <1行>
✅ 合格条件（採点表）: <観測可能な条件。収束タスクなら threshold/max も>
🛠 進め方: <直実行 / 計画 / ループ / Workflow / 並列エージェント>＋使うスキル

依頼文を書くこと＝採点表を書くこと。採点設計の精度がループ品質のほぼ全てを決める。

採点軸の作り方（3手順）

  • ① あいまいな形容詞を書き出す（良い／速い／読みやすい）
  • ② 数えられる事実に翻訳（読みやすい→「1文平均60字以下」）。「平均」か「各文上限」かを明確に。
  • ③ 各項目に配点（例：4項目×25点＝100点）。
翻訳できない質（構成が結論先行か・訴求が刺さるか）は捨てずLLM採点者へ観点として渡す。
収束タスクは threshold 90 / max 3 を1行で宣言。100点は求めない（粗探しが止まらず収束しない・Opusは90-92が落としどころ）。
3

Run（自律実行）

  • 既存スキルを「手」として選ぶ。実行形態を依頼規模で選ぶ。
  • 停止ゲート（ここだけ必ずユーザー確認）：消す/元に戻せない操作、本番反映、秘密・認証操作、cwd外の編集、大きな設計分岐。
  • 上限：ツール呼び出しや時間が嵩んだら、突き進まず報告して指示を仰ぐ。
  • ドリフト対策：内側ループは毎回もとの依頼文を先頭に再注入。指標を満たすだけの抜け穴に最適化していないか警戒。
  • ユーザー規約を尊重：判断材料はHTML、デプロイ規約、質問前ログ確認、「やる気管理しない」等。
4

Verify（検証）

  • 「完了」と言う前にテスト実行や実物確認で証拠を取る。
  • 自己採点しない（最重要）。採点は作った頭と別の頭にやらせる。AIの3クセ：①自己申告は完了の証拠でない ②期待に迎合し逆算で正当化 ③会話が長いほど判断が鈍る。

採点する係の4契約

  • ① まっさらな別の頭(fork)で採点（作る係の会話・言い訳を見せない）
  • ② 採点基準を書き換えさせない（裏口を塞ぐ）
  • ③ 絶対評価（前回比でなく依頼と採点軸への適合）
  • ④ 報告でなく実物を自分で開いて確かめる（wc -m で実測等）

採点は構造化して返す：score ／ quality.breakdown（軸ごとの点・キーは周回固定）／ feedback（次周の直し方）／ passed。overall は単純平均でない。

内側ループ（収束タスクの反復）

  • ① もとの依頼文＋採点表を先頭に置く（ドリフト対策）
  • ② 作る／直す（前回の指摘箇所だけ）
  • ③ 別の頭(fork)で独立採点し、score＋breakdown＋直し方を受け取る
  • ④ score ≧ threshold で抜ける。未達なら③の直し方を握って②へ
止め方のダイヤルは threshold と max の2つだけ。max 到達／3周連続同点（堂々巡り）で打ち切り。点が出るのに良くならない時は、ダイヤルでなく採点軸を直す。
最良版を採用：出力は確率的。各周の最良版を別に記録し、最後は最新でなく最良版を採る。
コストは「1周＝AI2回課金」（作る係＋採点する係）。
