82-84を狙って転のコマを1枚だけ作り直したら、スコアは 80 → 77 に後退しました。採点者の指摘が決定的でした。

「凪の髪色が p1(明) → p4・p6(暗) → p7(明) でぶれる」

つまり前回P6だけ直した結果、不整合が別ページへ移動しただけ。しかもこれは教訓台帳 L-21 が既に禁じていた行為でした ──「キャラ一貫性は生成の型で担保する。全該当コマを同一記述で一括生成する。バラバラの時期に部分再生成しない」。答えは最初から書いてあり、私がそれを破りました。

直したのはエピソードでなく「ハーネス」

  • injectCharacterAnchors() を実装 — characters[].visual_anchor を生成時に全パネルプロンプトへ機械注入（冪等・テスト付き）。手作業の逐語コピーは必ず漏れるので、SSOTを1箇所に集約して機械が配る。
  • アンカーに「ぶれる軸」を明示固定 — 髪の長さだけでなく色まで（JET-BLACK / never blonde / never short-haired）。ドリフトは"書かれていない属性"に必ず出る。
  • 手書きの人物描写を全行から剥がした — 手書きがあると注入がskipされSSOTが二重化するため。
  • 凪が写る11コマを一括再生成（L-21の処方を実行）。
  • 絶対ルールとして AGENTS/CLAUDE に明文化＋L-29として台帳化。テスト 35 green / QA PASS。
